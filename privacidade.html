Política de Privacidade

Versão 1.0 — Última atualização: Abril de 2026

1. Responsável e Operador dos Dados

O SurgApp actua como operador de dados nos termos da Lei Geral de Proteção de Dados (LGPD — Lei 13.709/2018). O controlador dos dados é o profissional de saúde titular da conta, que determina a finalidade e o conteúdo dos registos.

O desenvolvedor da aplicação trata os dados exclusivamente para prestar o serviço contratado, sem os utilizar para fins próprios, publicitários ou analíticos.

2. Dados Coletados

O aplicativo recolhe e armazena, mediante consentimento expresso do utilizador:

Dados do utilizador (profissional):

  • Endereço de e-mail (identificação de conta)
  • Configurações e preferências da aplicação
  • Listas personalizadas (hospitais, convênios, membros da equipa)

Dados de registos cirúrgicos:

  • Data, procedimento, hospital e convênio
  • Equipa cirúrgica (cirurgião, auxiliares, instrumentador, anestesista)
  • Informações financeiras do procedimento (valores, forma de repasse)
  • Observações clínicas e diagnóstico
  • Materiais utilizados (OPME)

Dados de pacientes:

  • Nome completo, data de nascimento, prontuário
  • Telefone de contacto e convênio
  • Histórico de procedimentos associados

Conteúdos multimédia:

  • Fotografias de procedimentos cirúrgicos e documentos (se a galeria estiver activa)
  • Termos de consentimento informado assinados digitalmente (se o módulo estiver activo)

3. Finalidade do Tratamento

Os dados são tratados exclusivamente para:

  • Prestação do serviço de registo cirúrgico ao profissional titular da conta
  • Geração de relatórios e exportações solicitadas pelo próprio utilizador
  • Sincronização entre dispositivos do mesmo utilizador
  • Processamento pontual de imagens para preenchimento automático de formulários (OCR)

Os dados não são utilizados para publicidade, análise de mercado, treino de modelos de IA próprios nem vendidos ou cedidos a terceiros.

4. Base Legal (LGPD)

O tratamento de dados pessoais e dados de saúde fundamenta-se em:

  • Consentimento expresso do titular (Art. 7º, I e Art. 11, I da LGPD) — recolhido na primeira utilização da aplicação
  • Legítimo interesse do controlador para prestação do serviço contratado (Art. 7º, IX)
  • Exercício regular de direitos em processo judicial, administrativo ou arbitral (Art. 7º, VI)

O consentimento pode ser revogado a qualquer momento através da eliminação da conta (Configurações → Sair do App → Eliminar Conta).

5. Armazenamento e Segurança

Os dados textuais são armazenados na plataforma Supabase, com:

  • Row Level Security (RLS) — cada utilizador apenas acede aos seus próprios dados
  • Transmissão sempre cifrada via HTTPS/TLS 1.2+
  • Servidores localizados na região South America (São Paulo) sempre que disponível
  • Autenticação por JWT com expiração automática

Os dados são também armazenados localmente no dispositivo do utilizador (IndexedDB) para funcionamento offline. Estes dados locais estão sujeitos às protecções do próprio sistema operativo do dispositivo.

6. Armazenamento de Fotografias

As imagens carregadas são armazenadas em bucket privado no Supabase Storage:

  • O bucket é configurado como privado — não acessível publicamente sem autenticação
  • O acesso às imagens é feito por URLs assinadas com validade limitada, geradas apenas para o utilizador autenticado
  • As imagens são isoladas por utilizador
  • Ao eliminar um registo ou a conta, as imagens associadas são também eliminadas

Recomendamos que o utilizador mantenha cópias locais das fotografias com relevância clínica, pois a disponibilidade do serviço de armazenamento depende de terceiros (Supabase).

7. Processamento por IA — OCR de Etiquetas

A funcionalidade de leitura ótica (OCR) de etiquetas hospitalares envia a imagem captada para a API Vision da OpenAI. Este processamento:

  • É realizado de forma pontual e sob pedido explícito do utilizador
  • A imagem é transmitida de forma segura (HTTPS) e processada em tempo real
  • A OpenAI não retém as imagens após o processamento, conforme os seus Termos de Serviço de API
  • O texto extraído é usado exclusivamente para pré-preencher o formulário — não é armazenado separadamente
  • A funcionalidade está sujeita a limite diário de utilizações

O utilizador pode optar por não utilizar o OCR e preencher manualmente. A funcionalidade pode ser desactivada nas Configurações.

8. Retenção dos Dados

  • Os dados são conservados enquanto a conta estiver activa
  • Após eliminação da conta, os dados são removidos dos servidores de forma imediata e permanente
  • Cópias de segurança de infra-estrutura (backups Supabase) podem conter dados por até 7 dias adicionais antes de rotação automática
  • Dados exportados pelo utilizador ficam sob sua responsabilidade exclusiva

9. Direitos do Titular (LGPD — Art. 18)

O utilizador tem os seguintes direitos, exercíveis directamente na aplicação:

  • Acesso: consultar todos os seus dados através do próprio app
  • Portabilidade: exportar dados em Excel ou JSON via Configurações → Exportar / Backup
  • Correcção: editar qualquer registo diretamente no app
  • Eliminação: apagar a conta e todos os dados em Configurações → Sair do App → Eliminar Conta
  • Revogação do consentimento: através da eliminação da conta
  • Informação: obter informações sobre o tratamento de dados através do contacto de suporte

10. Suboperadores e Terceiros

O SurgApp utiliza os seguintes serviços de terceiros:

  • Supabase Inc. (base de dados, autenticação, armazenamento de ficheiros) — dados processados na UE/EUA com cláusulas contratuais padrão (SCCs)
  • OpenAI LLC (processamento OCR de imagens — apenas quando a funcionalidade é utilizada) — processamento nos EUA sob Termos de API OpenAI

Nenhum dado é partilhado com outros terceiros para fins comerciais, publicitários ou analíticos.

11. Transferência Internacional de Dados

Os dados podem ser processados fora do Brasil (Supabase — EUA/UE, OpenAI — EUA). Estas transferências são realizadas com salvaguardas adequadas, incluindo cláusulas contratuais padrão e certificações de conformidade, em conformidade com o Art. 33 da LGPD.

12. Cookies e Rastreamento

O SurgApp não utiliza cookies de rastreamento nem ferramentas de análise comportamental de terceiros. O armazenamento local (IndexedDB, localStorage) é utilizado exclusivamente para funcionamento da aplicação.

13. Alterações à Política

Esta Política de Privacidade pode ser actualizada. Alterações substanciais serão comunicadas através da aplicação antes de entrarem em vigor, com possibilidade de o utilizador rever e aceitar as alterações.

14. Contacto — Encarregado de Dados (DPO)

Para exercer direitos LGPD, comunicar incidentes de segurança ou esclarecer dúvidas sobre o tratamento de dados, contacte o suporte através das Configurações da aplicação ou pela página de suporte.

Reclamações à autoridade competente podem ser dirigidas à Autoridade Nacional de Proteção de Dados (ANPD) — www.gov.br/anpd.

Esta Política de Privacidade é parte integrante dos Termos de Uso do SurgApp.
